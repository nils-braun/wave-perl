FLAGS

$DEBUG

Internal flag for debugging.

$WAVE_EXE

Internal path to the wave executable. Must not be changed!

$RESULT_DIR

Folder for the results. Will be created if not present.

$WORKING_DIR

Folder for the temporary files. If not specified, WAVE.pm will use a temporary folder. See $flagDeleteTempDir to delete the WORKING_DIR after clculations or not (especially important of the folder is not a temporary folder!)

$PARTICLE_FILE

Path to a file with WAVE-like particles. Is used when $flagOwnParticleFile = 1. If bunch is set you have to provide a PARTICLE_FILE or call make_particles.

$MAGNET_FILE

Path to a file with WAVE-like magnet data. Must be specified when using magnet mode file or magnet mode fileLinear. If provided, it will be used and the other magnet modes are overwritten.

$SUFFIX

SUFFIX is used as a suffix for all output files and messages.

$USER_TEXT

USER_TEXT is printed into all output files of provided.

$flagWriteTrajectory

Flag to write the trajectory to the RESULT_DIR.

$flagWriteSpectrum

Flag to write the spectrum to the RESULT_DIR.

$flagWriteLog

Flag to write the log to the RESULT_DIR/log.

$flagDeleteTempDir

Flag to delete the WORKING_DIR after calculation. Attention if the WORKING_DIR is provided.

$flagOwnParticleFile

Flag to set if the provided PARTICLE_FILE should be used.

$flagExitOnWaveError

Flag to set if WAVE.pm should exit on a WAVE error or not (usefull in script mode).

$flagWriteRoot

Flag to write all the results into a root file in the RESULT_DIR. Makes the calculations very slow.

$particleData

Content of the newly created or provided particle file after the calculation.

calc()

Use the input parameters and falgs defined before and start the calculation of WAVE. The errorLog and the log are written into files in the log folder which is a subfolder of the WAVE::RESULT_DIR. They are named after the pid of the WAVE process which can be seen in the log of WAVE.pm. After the WVAE process has finished, the WAVE output files are written into readably format and copied into WAVE::RESULT_DIR. They are named SUFFIX_filename.ending.

After the calculation, the following variables can be used for further analysis: @resultEndposition, @resultEndmomentum, $resultMaximumEnergy, $resultMaximumIntensity.

rewriteFiles

Internal function. For comments (in german) see WAVE.pm.

make_particles(BETAX, ALPHAX, BETAY, ALPHAY, EPSILON)

Instead of providing a $PARTICLE_FILE with the flag $flagOwnParticleFile you can creaty particle on the fly with this functions. Before calling the function, the number of particles must be set using the function setBunch(NUMBER, BUNCHMODE) in waveSub.pm. The twiss parameters (BETAX, ALPHAX, BETAY, ALPHAY, EPSILON) are used to control the shape of the particle distribution via a gaussion distribution. The input parameters are all in meter, without a unit or in meter rad.
